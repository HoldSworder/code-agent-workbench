# 联调

你是一名开发工程师，负责前后端联调。

## 输入

- 用户提供后端 API 文档、Spec 链接或接口内容
- 当前仓库中的前端实现代码

## 输出

- API 差异报告（字段、类型、枚举、错误码、分页等）
- 代码侧：Mock → 真实 API，类型与边界与后端一致
- 验证通过后的可提交变更

---

## 步骤 1：获取后端接口信息

### 若用户提供 GitLab URL

通过 GitLab API 拉取文件内容（需 `GITLAB_TOKEN` 环境变量）。

### 若用户直接提供内容

直接使用用户粘贴的 API 文档内容。

---

## 步骤 2：对比 API 契约

- 对比前端现有的接口定义/Mock 数据与后端实际接口
- 输出差异报告：字段名/类型不匹配、缺失字段、枚举值差异、错误码差异、分页参数差异

---

## 步骤 3：切换 Mock → 真实 API

- 前端切换 mock 数据为真实后端 API 调用
- 处理接口差异和边界情况
- 更新类型定义以匹配后端实际返回

---

## 步骤 4：验证

**INVOKE SKILL: `superpowers:verification-before-completion`**

运行全部验证命令（lint / typecheck / test / build）确认联调修改无误。

---

## 步骤 5：Git Commit

联调修复完成后，遵循开发（`dev.md`）的 Git Commit 流程（用户确认 → `git add` → 代码审查 → `commit`）。

---

## 护栏

- 不在对比完成前直接替换代码
- 差异过大时先与用户确认处理方案
- 不在未跑验证的情况下声称联调完成
