# 设计稿获取

当用户提供了 Figma 链接或明确要求对齐设计稿时，通过 Figma MCP 拉取设计信息，形成可实现、可验收的 UI 规格约束。

## 输入

- Figma 文件链接或节点标识（以用户提供的为准）
- 本期需求对应的页面/组件范围说明
- 需求采集阶段产出的结构化需求摘要（用于对齐「改哪里」）

## 输出

**UI 规格约束（UI specification constraints）**，包括：

1. 通过 Figma MCP 获取的节点树、布局、组件层级及与需求相关的视觉规格：**颜色、字体、字号、间距、圆角** 等可落地 token 或具体值
2. **局部变更约束**：仅针对用户指定范围给出实现约束；不得因「对齐设计稿」推断全页重绘或扩大改动面
3. 前端实现须 **严格遵守** 上述与本期需求相关的设计规格（与工程现有 design token 冲突时，在后续阶段显式列出待决策项）

若无 Figma 或用户未提供设计稿，本阶段产出可为简短说明「无设计稿输入」，不阻塞后续需求对齐（除非用户声明必须有稿）。

## 护栏

- 调用 Figma MCP 前须遵循项目内 Figma 相关 skill 要求（如有 `figma-use` 等前置约束）
- 禁止为美观擅自扩展修改范围到未授权区域
- 设计信息拉取失败时须说明原因与补救方式（重试、换链接、手工标注），禁止编造节点数据
