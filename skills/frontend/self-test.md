# 代码 Review

对当前仓库运行完整的质量验证流水线，确保代码在合并前处于健康状态。

## 输入

- 当前工作区代码（开发 / 联调 完成后或任意自检节点）
- 项目根目录的 `package.json` 中定义的 scripts（`lint` / `typecheck` / `test` / `build`）

## 输出

- 各命令的实际执行结果（完整输出保留，尤其是失败信息）
- 以表格汇总的步骤状态（见下）
- **可选（未来）**：由 Codex 或其它 Agent 做交叉跑测 / 二次验证时，在本阶段说明中记录约定与入口；当前仍以本机 `pnpm` 流水线为准

---

## 执行步骤

依次在终端执行以下命令，**每一步都必须执行**，不得跳过：

### 1. Lint 检查

```bash
pnpm lint
```

若 lint 命令不存在或项目未配置，记录为「跳过」并继续。

### 2. 类型检查

```bash
pnpm typecheck
```

若 typecheck 命令不存在，尝试 `pnpm tsc --noEmit`。若均不可用，记录为「跳过」并继续。

### 3. 单元测试

```bash
pnpm test
```

若 test 命令不存在或项目未配置测试，记录为「跳过」并继续。

### 4. 构建

```bash
pnpm build
```

构建必须成功。若失败，分析错误并尝试修复后重试。

---

## 输出格式

以表格形式汇总各步骤结果：

| 步骤 | 状态 | 备注 |
|------|------|------|
| Lint | ✅ 通过 / ⚠️ 跳过 / ❌ 失败 | 具体信息 |
| 类型检查 | ✅ / ⚠️ / ❌ | |
| 单元测试 | ✅ / ⚠️ / ❌ | 通过/失败数 |
| 构建 | ✅ / ❌ | |

---

## 护栏

- 不得在未实际运行命令的情况下声称验证通过
- 构建失败时，尝试修复后必须重新运行验证
- 所有命令的完整输出必须保留，不得截断错误信息
- **Codex 交叉测试** 若启用，须与上述流水线结论一致后再进入 E2E 浏览器测试（`e2e-test.md`）；未启用时不杜撰「已交叉验证」
